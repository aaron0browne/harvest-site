Clone from GitHub recommended

Keep up with bug fixes and new features incrementally by starting your project as a cloned repository from the Harvest git repository. Why?

  • Any upstream changes we make for various bug fixes and new features can be easily applied to your Harvest build with minimal or no effort
  • You start off immediately versioning your code which is a best (necessary) practice for software development
  • This is not only for bleeding edge usage of Harvest. Using this method, you can choose which version (old or new) of Harvest you want to use.

Follow these simple steps:

  1. Create a new empty project 
    git init my-harvest-project
cd my-harvest-project
  2. Add the upstream repository and fetch it 
    git remote add upstream git://github.com/cbmi/harvest-template.git
git fetch upstream
  3. Merge the lastest version into your local repository. If you prefer a tagged release (the master branch is very stable), you can replace master with the lastest tag. 
    git merge upstream/master
  4. Add the origin server (where you want to push your changes to) 
    git remote add origin YOUR_REMOTE_URL
git push -u origin master
