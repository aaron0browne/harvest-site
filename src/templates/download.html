Prerequisites

Harvest requires Python 2.6 or 2.7. Check to see which version of Python you have:

python --version
Required stdlib modules. Harvest depends on the SSL and SQLite modules to be built in the standard library. If you are building Python from source, the OpenSSL and SQLite C headers must be installed prior to building Python. On CentOS, these are the sqlite-devel and openssl-devel packages.

In order to install the Harvest package, Pip or setuptools must be installed. Check for either command.

which pip easy_install

If nothing prints out, then do the following to get Pip (the preferred installer). Note, you may have to do sudo python if you are installing it globally on your system.

curl http://python-distribute.org/distribute_setup.py | python
curl https://raw.github.com/pypa/pip/master/contrib/get-pip.py | python
Watch out for sudo! The root user $PATH most likely does not include /usr/local/bin. If you did not install Python through your distro's package manager, use the absolute path to the new Python binary to prevent installing the above libraries with the wrong version (like Python 2.4 on CentOS 5), e.g. /usr/local/bin/python2.7.

Install

The Harvest package is availble on PyPi and can be installed using Pip or easy_install. Again, you may need to prefix this with sudo if this is being installed globally on your system.

Pip 
pip install harvest
easy_install 
easy_install harvest

Setup A New Project

Harvest comes with a command-line tool that sets up new projects with one command.

harvest init myproject

Install the Demo

The demo can also be installed with one line.

harvest init-demo openmrs

For full details on the available commands and options run harvest --help or for help with a specific command, run harvest [command] --help. The command documentation is also available on the Harvest Github pages.
